# BLUF Conventions — Condensed Reference

BLUF ("Bottom Line Up Front") is the compressed, single-document cousin of the Minto pyramid — see `minto-pyramid.md` for the full structure this is derived from. Use this version for memos, briefs, and status reports where a full pyramid build-out would be overkill but conclusion-first discipline still matters.

This reference is the canonical full version. Other plugins' BLUF skills (e.g. `transformation:tech-strategy-brief`, `performance:performance-narrative`, `pmo:status-report`) carry a condensed inline version of this shape so each plugin works without depending on `consulting` being installed.

## The shape

1. **Bottom line** (1-2 sentences, always first): the recommendation, decision, or headline result. Not the topic — the answer.
2. **Why** (2-4 points): the MECE reasons supporting the bottom line. Same discipline as a Minto key line, just shorter.
3. **What this means / what's needed** (if applicable): the ask — a decision, a resource, a sign-off.
4. **Detail** (everything else): background, methodology, caveats, data — for the reader who wants to go deeper, not required to get the point.

## The test

A reader who reads only the first 1-2 sentences should know the answer. A reader who reads only the first paragraph should know the answer *and* the main reasons. Everything after that is optional depth, not required reading.

## Common failure modes to flag

- **Leading with context instead of the bottom line** — "Over the past quarter, we have been monitoring..." before the actual headline.
- **Burying the ask** — a status update that never says what decision or resource is actually needed.
- **Detail creeping above the fold** — methodology or caveats appearing before the bottom line is established.
- **No bottom line at all** — a chronological narrative of what happened, with the reader left to infer the takeaway.

## Quick self-check

1. Does sentence one state the answer, not the topic?
2. Could a reader stop after the first paragraph and still act correctly?
3. Is the ask (if any) explicit and impossible to miss?
